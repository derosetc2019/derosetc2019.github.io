9.  Deployment


9.1 Peparing for deployment
 

  1. Maak een commit in je huidige (wwarschijnlijk master) branch.
    1. git add --all
    2. git commit -m "Commit voordat we beginnen aan heroku"
    3. git status (geeft de message nothing to commit, workingdirectory clean)
       
  2. Om ervoor te zorgen dat we altijd terug kunnen naar de oude situatie als al de onderstaande handelingen fout gaan maak je een nieuwe branch heroku1 met het onderstaande commando
     
    1. git checkout -b heroku1
       
  3. Check met git branch of je nu op de heroku1 branch zit.
  4. log in met:
     
    1. heroku login
       
  5. Maak een Procfile met:
    1. echo "web: vendor/bin/heroku-php-apache2 public/" > Procfile
       
  6. Maak een comit met
    1. git commit -m "Initial Procfile"
       
  7. Maak een App met:
     
    1. heroku create.
       
  8. Er is nu ook een remote gemaakt (git remote -v show) en op de heroku site (ga naar de site en checlk dit) een nieuwe app. Je kunt nu ook naar de basisapp van heroku kijken via de gegeven url. Er is ook een git repository gemaakt op heroku.
     
  9. Maak een APP_KEY aan met:
    1. heroku config:set APP_KEY=$(php artisan --no-ansi key:generate --show)
    2. . De APP_KEY staat nu ook in de config-vars op de heroku site
       
  10. Push nu de lokale branch naar via de zojuist bij stap 5 gemaakt remote naar de master branch op heroku met: 
    1. git push heroku heroku1:master
    2. heroku is hier mijn remotelink https://git.heroku.com/tranquil-beyond-71717.git afgekort mijn remote
    3. heroku1 is mijn branch waar ik op zit.
       
  11. Wanneer het niet is gelukt is het waarschijnlijk je Procfile. Verander van UTF-16 LE naar UTF-8 in Visual Studio code. Update ook even je composer-lock.json file met het commando composer update. Vergeet niet na beiden aanpassingen een commit te maken en push weer met
    1. git push heroku heroku1:master
    2. Zie ook https://help.heroku.com/CH22EKEK/why-am-i-seeing-push-failed-can-not-parse-procfile-when-deploying-my-app.
    3. Ook deze link kan je verder helpen https://github.com/composer/composer/issues/8465
       
  12. Open de site door ctrl + click uit te voeren op de url in je terminaltekst gegenereerd door het commando van stap 11.1 hierboven
     

9.2 Databases

  1. We maken een postgresql database aan met:
    heroku addons:create heroku-postgresql:hobby-dev --remote heroku
    Let op --remote heroku, mijn remote heet namelijk heroku. Je kunt de naam van jouw remote vinden met het commando git remote -v show. 
  2. Bij de Config-Vars op de heroku site is nu een DATABASE_URL property erbij gekomen
  3. Maak nu bij de Config Vars op heroku site een nieuwe DB_CONNECTION=pgsql
    property  - value paar.
  4. Pas config/database.php aan
  5. Pas database/migrations/create_hotel_tables.php aan  (verwijder bij schema 'rates' de index('room_type_id')). Dit moet gedaan worden omdat we een pgsql database hebben gemaakt en geen mysql database
  6. Maak een commit
  7. Doe een migration bij heroku met:
    heroku run php artisan migrate:fresh --seed --force --remote heroku
  8. Ga naar de site en probeer je te registreren. Je krijgt dan een 500 | server error
  9. We moeten nog een mailserver maken die onze mail opvangt. Er is niet een gratis mailserver die je zonder dat je creditcardgegevens moet opgeven kunt gebruiken bij heroku. We kunnen wel een ander mailserver gebruiken mailtrap. Ga naar https://mailtrap.io/ en maak een account aan.
  10. Log in bij mailtrap en ga naar je inboxes => myinbox => smtp-settings =>integrations => kies uit de select laraval. De onderstaande gegevens zijn mijn gegevens dus die kun je niet gebruiken maar je moet je eigen gegevens in de Config Vars zetten van heroku. Ga dit nu doen.
    MAIL_MAILER=smtp
    MAIL_HOST=smtp.mailtrap.io
    MAIL_PORT=2525
    MAIL_USERNAME=f9073905341239
    MAIL_PASSWORD=e62796bb121235
    MAIL_ENCRYPTION=tls
  11. Ga terug naar de site en test of het registreren nu wel lukt. Het registratiemailtje vind je bij mailtrap.io
  12. We moeten nu nog even afvangen dat iemand zijn email heeft verifieerd. Daar kan je de onderstaande method voor gebruiken
    https://laravel.com/api/8.x/Illuminate/Auth/MustVerifyEmail.html#method_hasVerifiedEmail
     
  13. Type bij de HomeController.php de onderstaande code en let op het uitroepteken op regel 25!!!!




     
  14. Maak een commit en push met je remote naar heroku
